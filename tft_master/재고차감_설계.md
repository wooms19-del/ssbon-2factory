# 재고 자동 차감 설계

목표: 생산하면 자재가 재고에서 자동으로 빠지게. 원육~포장재 전부.
이달 말 재고조사로 품목별 재고수량이 들어오면 그 위에 얹어 실시간 재고 운영.

## 핵심 원칙 (관리자 확정)
1. **반제품 재료는 합산 1회** — 같은 반제품을 여러 제품으로 나눠 포장해도(예: 130g 반제품 1000개 → 일반 500 + 마트용 500) 재료는 ea 합산으로 한 번만 차감. 이중 차감 방지.
2. **포장재는 제품별** — 외박스·트레이는 제품마다 다르므로 각각 차감.
3. **원육은 파쇄 단계에서** — 부위별로.
4. **차감 방식은 실시간 + 정산 둘 다** — 현재고 상시 표시 + 재고조사 기준일 정산.

## 결정적 발견: 모든 자재가 실측값으로 기록됨
레시피 계산이 거의 불필요. 현장이 실사용량을 이미 다 입력하고 있음.

| 자재 | 실측 필드 | 출처 컬렉션 |
|---|---|---|
| 소스 | `sauceKg` | packing (내포장) |
| 파우치 | `pouch` | packing |
| 메추리알(부재료) | `subKg` | packing |
| 포장재(박스·트레이·인박스) | `materials[].actual` | outerpacking (외포장) |
| 박스 수 | `outerBoxes` | outerpacking |
| 트레이 | `trayUsed` | outerpacking |
| 파쇄육 산출 | `kg` (부위 `type`) | shredding (파쇄) |
| 원육 투입 | `kgIn` (부위 `type`) | shredding |

→ 재고 차감 = "기록된 실사용량을 재고에서 빼기". 레시피 계산 아님.
→ 마스터 레시피의 역할: 자재 재고 ↔ 실적 필드를 잇는 '지도' (수량은 실적에서).

## 기존 재고 기능 (이미 있음 — js/stock.js)
입고관리가 이미 구축됨. 여기에 '생산 차감'을 이어붙임.
- `stockIn` (2공장 입고), `stockIn_f1` (1공장), `stockIn_ca` (천안물류)
- `transfer` (공장 간 이동)
- 서브탭: 2공장/1공장/천안물류

재고 = 입고(stockIn) − 생산차감(신규) − 출고(shipment, 완제품)

## 차감 구조
```
[생산 실적]                  [재고 차감]
packing.sauceKg      →  소스 재고 −
packing.pouch        →  파우치 재고 −
packing.subKg        →  메추리알 재고 −
outerpacking.materials.actual → 포장재 재고 − (제품별)
shredding.kgIn (type별) → 원육 재고 − (부위별)
shredding.kg (type별)   → 파쇄육(반제품) 재고 +/−
```

## 재고조사 기준일 처리
- 이달 말 재고조사 → 품목별 실사 재고수량 입력
- 그날을 기준점으로: 현재고 = 실사값 − (기준일 이후 생산차감)
- `stockReg` 플래그(이미 outerpacking에 있음) 활용해 '차감 반영됨' 관리

## 남은 작업 (재고조사 값 나오면 구현)
1. 자재 재고 컬렉션 설계 (item_code 기준, 마스터 연동)
2. 생산 실적 → 자재 차감 로직 (실측 필드 매핑)
3. 재고조사 기준일 세팅 UI
4. 실시간 현재고 화면 (입고 − 차감 − 출고)
5. 검증: 실측 차감이 정확한지 실제 생산일로 대조
