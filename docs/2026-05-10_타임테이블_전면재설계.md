## 작업 개요

## 1. 시뮬레이션 엔진 — 동적 계산

### 핵심 원칙
모든 공정 X 종료 = max(자체 처리 종료, 앞 공정 Y 종료 + Y 마지막 산출이 X 통과 시간)

### 인원 자동 계산
```
파쇄 풀가동 (13:30~) = 총원 - 내포장(6) - 이송(2) - 관리(2) - 제수(N)
파쇄 점심후 (12:30~13:30) = 풀가동 - 4
```

사용자는 총원 + 제수만 입력 → 파쇄 인원·시간 자동 변경.

### 점심 시간대 동적 가용 인원
```
11:30~12:30 (점심 1차):
  · 전처리가 11:30 전 끝났으면 → 전처리조가 파쇄로 합류
  · 전처리 진행 중이면 → 파쇄 0명

12:30~13:30 (점심 2차):
  · 후공정조 복귀 + 이송 합류 → 파쇄 = wkCrush + wkTrans
  · 전처리조 점심

13:30~ (풀가동):
  · 파쇄 = wkPackPeak
```

이전 모드 A/B/C (한꺼번에/교대/동시)는 임의 가정이라 제거. 동적 계산으로 통합.

### 레토르트 = 3대 + 대차 8개 한도
- 설비 3대 동시 가능
- 대차 8개 → 회차당 4대차 → 동시 최대 2회차 (4+4=8)
- 회차 끝나야 대차 회수 → 재투입

### 자숙 호별 파쇄 처리 시간 (FIFO)
- 자숙 1호 426kg: 와건 11:42 → 13:42 (120분, 14명)
- 자숙 2호 414kg: 13:45 시작 → 15:14 (89분, 18명)

## 2. UI 시각화

### 화면 구성
1. 결론 박스
2. 인원 운용 전략 (자연어)
3. 공정 타임라인 (좌, 1) + 시간대별 인원표 (우, 1)
4. 각 공정이 왜 그 시간인지 (5개 카드 그리드)
5. 계산 보고서 표

### 타임라인
- 각 공정 행 분리, 막대마다 인원·kg 표시
- 호버 시 검은 툴팁 (시각/인원/처리량/계산식)
- 점심 영역 노란 음영
- 파쇄 = 자숙 호별 분할 (호별 색상 다름)
- 점심 행 추가

### 인원표
- 11개 컬럼 모두 균등 9.09% (table-layout:fixed)
- 행 균등 분배 (height: 100/N%)
- 셀 호버 시 노란색 강조
